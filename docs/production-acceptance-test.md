# 本番受入テスト

本アプリを学内業務で使用開始する前に、担当職員が以下を確認する。

## 1. データ取込

- [ ] 配布用Excelテンプレートをダウンロードできる
- [ ] 共通情報と申請人情報を入力したExcelを読み込める
- [ ] 空欄、誤った性別、誤った日付、重複番号などがエラーとして表示される
- [ ] エラーのある行が申請人一覧に残り、修正対象を確認できる
- [ ] 有効な申請人だけが書類作成対象になる

## 2. 招へい理由書

- [ ] 正式様式の背景が表示される
- [ ] 書類作成日、提出先公館、招へい人、担当者情報が正しい
- [ ] 氏名、国籍、職業、性別、生年月日、年齢が正しい
- [ ] 公文書番号が指定位置に表示される
- [ ] 長い氏名、住所、プログラム名でも文字切れや重なりがない
- [ ] 個別PDFと全員分ZIPを開ける

## 3. 身元保証書

- [ ] 正式様式の背景が表示される
- [ ] 申請人情報が正しい
- [ ] 保証人の所属・肩書・氏名が正しい
- [ ] 保証人職業が空欄でも出力できる
- [ ] 保証人生年月日から書類作成日現在の年齢が正しく計算される
- [ ] 公館種別の未選択、大使館、総領事館がそれぞれ正しく表示される
- [ ] 1申請人につき1通のPDFが作成される
- [ ] 個別PDFと全員分ZIPを開ける

## 4. 一括出力

- [ ] 選択中の申請人について2書類ZIPを作成できる
- [ ] 全申請人について招へい理由書ZIPを作成できる
- [ ] 全申請人について身元保証書ZIPを作成できる
- [ ] 「全員の全書類ZIP」で申請人別フォルダが作られ、各フォルダに2通入る
- [ ] 20名分でブラウザが停止せず、ZIPを正常に開ける
- [ ] ファイル名に使用できない文字が安全な文字へ置換される

## 5. 印刷・目視確認

- [ ] Chrome最新版でA4・倍率100%で印刷する
- [ ] 招へい理由書を実際の提出済み見本と重ねて、文字位置を確認する
- [ ] 身元保証書を実際の提出済み見本と重ねて、文字位置を確認する
- [ ] 文字の欠落、文字化け、枠外表示、チェック位置のずれがない
- [ ] PDFをAdobe Acrobat Readerでも開いて確認する

## 6. 個人情報・運用

- [ ] 実データは担当者の管理されたPCでのみ扱う
- [ ] 共用PCでは使用しない
- [ ] 作業終了後にブラウザを再読み込みまたは閉じ、表示データを消す
- [ ] ダウンロードしたExcel、PDF、ZIPを所定の保存場所へ移し、不要なコピーを削除する
- [ ] 本番URLを検索エンジンに登録しない
- [ ] 本番URLの共有範囲とアクセス制御を部署内で決定する

## 合格条件

- 自動検証、TypeScript確認、Production buildがすべて成功している
- 上記チェック項目がすべて完了している
- 招へい理由書と身元保証書について、担当職員1名以上が実際の提出見本との位置照合を完了している
- 実データで作成したPDFを提出前に別の職員が確認する運用を決めている
